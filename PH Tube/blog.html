VAR:

  • Scope: Variables declared with var are only accesible within the function where they are defined. So they are function-scoped.
  • Hoisting: Variables declared with var are moved to the top of their scope (but not their initialization) before code execution. They will be initialized with undefined.

LET:

  • Scope: Variables declared with let are only accesible only within the block where they are defined. So they are block-scoped. A block can be a function, loop or any code enclosed within {}.
  • Hoisting: Variables declared with let are also moved to the top of their scope (but not their initialization) before code execution. But they will not be initialized so you cannot use them.

CONST:

  • Scope: It is also a block-scoped like let
  • Immutability: Variables declared with const are meant to be constants so we cannot reassign it's value after initializaton. But for objects and arrays declared with const, we can still modify the properties or elements; but can't reassign the variable itself.

NULL:

  • Absence Of Value: It represents the intentional absence of any object value or value in general.
  • As a Placeholder: It is used to indicate that a variable has not been initialized yet.
  • Returned by Functions: It is returned by functions when a it did not find a meaningful result.

UNDEFINED:

  • Default Value: Variables declared without a value are initialized with the default value of the data type.
  • Property Access: Trying to access an object property which does not exists will gives undefined as an output.
  • Array Elements: Trying to access an array element beyond the current length of the array this will give undefined as an output.
  • Function Return: Functions that do not return a value will return undefined.

REST API:

A REST API is a way for different software applications to communicate over the internet by following a set of rules. These rules are based on the REST architectural style, which ensures that the communication is straightforward and efficient. REST APIs allow programs to send and receive data using simple web addresses, making it easy to request and share information. For example, a client application might ask a server for some data, like a list of users, by sending a request to a specific URL. The server then responds with the requested data, often in a format like JSON. This method of communication helps different programs work together seamlessly over the web, enabling them to share data and functionality easily.
